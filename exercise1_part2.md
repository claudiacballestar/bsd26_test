## Description of Git commands


- git config: to configurate a user and a user email
- git init: to initializate the folder into the git environment
- git commit: to save (write) the given documents in a fixed way
- git status: check the actual status of the reposistory, last changes, what was added...
- git add: to save the given documents in a "in between" stand
- git log: history of commits
- git diff: checks the difference between the current file and the file in their last commited stage
- git pull: download content from GitHub to the current folder
- git push: upload content of the git folder to GitHub
- git branch: creates a new branch to work with (as a virtual subdirectory, to work in changes without modifying what was originally there)
- git checkout: change between branches (checkout old, write the new name to change into)
- git merge: actualize changed files from a branch, like the different stages. Not always direct: sometimes it has to be checked in advance what changes have "priority"
